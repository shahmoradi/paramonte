Detailed Description

Generate and return .true. is the input path is an extant system directory, otherwise return .false..

The procedures under this generic interface utilize the Fortran intrinsic inquire() statement to infer the existence of the specified input path as a system directory.

Parameters
[in]path: The input scalar character of kind any supported by the processor (e.g., SK, SKA, SKD , or SKU) containing a POSIX-style or Windows-style path.
[out]iostat: The optional output scalar integer of default kind IK.
  1. If present and no error occurs, it is set to 0 on output.
  2. If present and an error occurs, it is set to a positive non-zero value.
  3. If missing and an error occurs, then the program halts by calling error stop followed by the relevant error message.
(optional.)
[in,out]iomsg: The input/output scalar character of default kind SK containing the error message, if any error occurs.
A length type parameter value of LEN_IOMSG is generally sufficient for iomsg to contain the output error messages.
(optional. It can be present only if the iostat output argument is also present.)
Returns
pathIsDir : The output scalar of type logical of default kind LK.
It is .true. if the input path is an extant system directory, otherwise .false..

Todo:
High Priority: The current Fortran standard 202x does not allow passing characters of non-default kind to the intrinsic Fortran statements and procedures.
As such, the implementation of this procedure for non-default character kinds leads to compile-time kind mismatch errors.
This procedure should be converted back to a generic interface in the future when non-default character kinds are also fully supported by the intrinsic functions.
Todo:
Critical Priority: This generic interface should be extended in the future to gracefully handle intrinsic inquire() exceptions.
