function verify ( )

Check the requirements of the installed ParaMonte MATLAB library.

This function takes no input arguments and returns nothing.


Possible calling interfaces ⛓

pm.lib.verify();


Example usage ⛓

1cd(fileparts(mfilename('fullpath'))); % Change working directory to source code directory.
2addpath('../../../'); % Add the ParaMonte library root directory to the search path.
3
4pm.lib.verify();

14At least one MPI runtime library installation from vendors

16
17The MPI runtime libraries are required ONLY for the parallel ParaMonte simulations.
18The MPI-parallel version of the ParaMonte library requires
19one of the following MPI runtime libraries:
20
21 1) Intel MPI runtime library (recommended and available only for Windows and Linux)
22
23 This can be downloaded from Intel MPI library website.
24
25 2) MPICH MPI runtime library (available only for Linux and macOS/Darwin)
26
27 This can be downloaded and installed via a package manager.
28 On Linux Ubuntu systems, you can use the shell installation command:
29
30 sudo apt install mpich
31
32 On macOS/Darwin systems, you can use the Homebrew application to install MPICH:
33
34 brew install mpich && brew link mpich
35
36 If you do not have Homebrew installed on your system, you can do so by
37 typing the following command in your (Bash/zsh) shell terminal:
38
39 /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
40
41 3) OpenMPI runtime library (available only for Linux and macOS/Darwin)
42
43Once you have installed any of the above MPI runtime libraries,
44ensure the `mpiexec` MPI executable binary is available on the command line in your terminal.
45On Windows systems with Intel MPI library, you may have difficulty finding mpiexec in Windows terminals.
46For this reason, Intel provides its dedicated Intel OneAPI command-prompt on Windows which has all
47environmental paths predefined within the shell.
48
49 On Windows systems, you can also directly run the Intel Windows `vars.bat` Batch script within a
50 compatible Windows shell to define all necessary environmental variables for MPI-parallel simulations.
51 By default, this script is frequently stored in the following installation path:
52
53 "C:\Program Files (x86)\Intel\oneAPI\mpi\latest\env\vars.bat"
54
55 On Linux systems, you can also directly run the Intel `vars.sh` Bash script within a
56 compatible terminal to define all necessary environmental variables for MPI-parallel simulations.
57 By default, this script is frequently stored in the following installation path:
58
59 "/opt/intel/oneapi/mpi/latest/env/vars.sh"
60
61For more information on running MPI-parallel simulations, visit:
62
63 <a href = "www.cdslab.org/pm">www.cdslab.org/pm</a>
64
65On Linux/macOS systems, launch the MPI-parallel ParaMonte
66simulation on the shell terminal via the following command:
67
68 mpiexec -n 3 matlab -batch "main"
69
70where,
71
72 1. ``mpiexec`` is the MPI launcher application.
73 2. ``-n 3`` specified three processes to be used for the MPI-parallel simulation.
74 3. ``matlab -batch`` launches the LANG_NAME software per MPI process on the command line.
75 5. ``main`` is the name of your script containing the ParaMonte library task to run in parallel.
76 Replace the script name with whatever the actual name of your script file is.
77 Notice the MATLAB filename `main` appears without the file extension.
78 Notice the MATLAB filename is enclosed with quotation marks.
79
80On Windows systems using the Intel MPI library, an additional ``-localonly`` flag is required:
81
82 mpiexec -localonly -n 3 matlab -batch "main"
83
84where,
85
86 1. ``-localonly`` indicates a parallel simulation on only a single node.
87 This flag will obviate the need for MPI library credentials registration.
88 For more information, visit the Intel MPI library website.
