function savefig ( in  varargin)

Export figures in a publication-quality format.

This function saves a figure or single axes to one or more vectors and/or bitmap file formats and/or outputs a rasterized version to the workspace, with the following properties:

  1. Figure/axes reproduced as it appears on the screen.
  2. Cropped/padded borders (optional).
  3. Embedded fonts (vector formats).
  4. Improved line and grid line styles.
  5. Anti-aliased graphics (bitmap formats).
  6. Render images at native resolution (optional for bitmap formats).
  7. Transparent background supported (pdf, eps, png, tif, gif).
  8. Semi-transparent patch objects supported (png, tif).
  9. RGB, CMYK, or grayscale output (CMYK only with pdf, eps, tif).
  10. Variable image compression, including lossless (pdf, eps, jpg).
  11. Optional rounded line-caps (pdf, eps).
  12. Optionally append to file (pdf, tif, gif).
  13. Vector formats: pdf, eps, emf, svg.
  14. Bitmap formats: png, tif, jpg, bmp, gif, clipboard, export to workspace.

This function is especially suited to exporting figures for use in publications and presentations because of the high quality and portability of media produced.

Note that the background color and figure dimensions are reproduced (the latter approximately, and ignoring cropping & magnification) in the output file.
For transparent background (and semi-transparent patch objects), use the -transparent option or set the figure 'Color' property to 'none'.
To make axes transparent set the axes 'Color' property to 'none'.
PDF, EPS, TIF, and PNG are the only formats that support a transparent background.
Only TIF and PNG formats support the transparency of patch objects.

The choice of renderer (opengl/zbuffer/painters) greatly impacts the output quality.
The default value (opengl for bitmaps, painters for vector formats) generally gives good results, but if you are not satisfied, then try another renderer.

Note
  1. For vector formats (EPS, PDF), only painters generate vector graphics.
  2. For bitmap formats, only OPENGL correctly renders transparent patches.
  3. For bitmap formats, only painters correctly scale line dash and dot lengths when magnifying or anti-aliasing.
  4. Fonts may be substitued with Courier when using painters.

When exporting to vector format (PDF & EPS) and bitmap format using the painters renderer, this function requires that ghostscript is installed on your system.
You can download this from: http://www.ghostscript.com
When exporting to EPS, pdftops from the Xpdf function suite are also required.
You can download this from: http://xpdfreader.com

SVG output uses MATLAB's built-in SVG export if available, or otherwise the fig2svg (https://github.com/kupiqu/fig2svg) or plot2svg (https://github.com/jschwizer99/plot2svg) utilities, if available.

Note
Cropping and padding are not supported in SVG and EMF output.
Parameters
[in]varargin: Any property, value pair of the following set:

  1. The option filename - string containing the name (optionally including full or relative path) of the file, the figure is to be saved as.
    If no path is specified, the figure is saved in the current folder.
    If no name and output arguments are specified, the figure FileName property is used.
    If this property is empty, then the default name 'export_fig_out' is used.
    If neither file extension nor a format parameter is specified, a ".png" is added to the filename, and the figure is saved in PNG format.

  2. The option -<format> - string(s) containing the output file extension(s).
    Options are '-pdf', '-eps', 'emf', '-svg', '-png', '-tif', '-jpg', '-gif', '-bmp'.
    Multiple formats can be specified without restriction.
    For example: savefig('-jpg', '-pdf', '-png', ...).
    Note that '-tif', '-tiff' are equivalent, and so are '-jpg','-jpeg'.

  3. The option -transparent indicating that the figure background is to be made transparent (PNG, PDF, TIF, EPS, EMF formats only). Implies -noinvert.

  4. The option -nocrop indicating that empty margins should not be cropped.

  5. The option -c[<val>,<val>,<val>,<val>] indicating crop amounts.
    It must be a 4-element vector of numeric values: [top, right, bottom, left].
    where NaN/Inf indicates auto-cropping, 0 means no cropping, any other value means cropping in pixel amounts. e.g. '-c7,15,0,NaN' Note that SVG and EMF formats do not support this option.

  6. The option -p<val> to pad a border of width val to exported files, where val is either a relative size with respect to the cropped image size (i.e. p=0.01 adds a 1% border).
    For EPS and PDF formats, val can also be integer in units of 1/72 points (abs(val)>1).
    The input val can be positive (padding) or negative (extra cropping).
    If used, the -nocrop flag will be ignored, i.e. the image will always be cropped and then padded. Default: 0 (i.e. no padding).
    Note that SVG and EMF formats do not support this option.

  7. The option -m<val> where val indicates the factor to magnify the figure dimensions when generating bitmap outputs (does not affect vector formats).
    Default: '-m1' (i.e. val=1). Note: val~=1 slows down savefig.

  8. The option -r<val> where val indicates the resolution (in pixels per inch) to export bitmap and vector outputs, without changing the dimensions of the on-screen figure. Default: '-r864' (for vector output only).
    Note: -m option overrides -r option for bitmap exports only.

  9. The option -native indicating that the output resolution (when outputting a bitmap format) should be such that the vertical resolution of the first suitable image found in the figure is at the native resolution of that image.
    To specify a particular image to use, give it the tag 'export_fig_native'.
    Notes: This overrides any value set with the -m and -r options.
    It also assumes that the image is displayed front-to-parallel with the screen.
    The output resolution is approximate and should not be relied upon.
    Anti-aliasing can have adverse effects on image quality (disable with the -a1 option).

  10. The option -a1, -a2, -a3, -a4 indicating the amount of anti-aliasing (AA) to use for bitmap outputs when GraphicsSmoothing is not available.
    '-a1'=no AA; '-a4'=max.
    Default: 3 for HG1, 1 for HG2.

  11. The option -<renderer> to force a particular renderer (painters, opengl or [in R2014a or older] zbuffer).
    Default value: opengl for bitmap formats or figures with patches and/or transparent annotations and painters for vector formats without patches/transparencies.

  12. The option -<colorspace> indicating which colorspace color figures should be saved in: RGB (default), CMYK or gray. Usage example: '-gray'.
    Note: CMYK is only supported in PDF, EPS and TIF formats.

  13. The option -q<val> to vary bitmap image quality (PDF, EPS, JPG formats only).
    A larger val, in the range 0-100, produces higher quality and lower compression. val > 100 results in lossless compression.
    Default: '-q95' for JPG, ghostscript prepress default for PDF,EPS.
    Note: lossless compression can sometimes give a smaller file size than the default lossy compression, depending on the image type.

  14. The option -n<val> to set minimum output image size (bitmap formats only).
    The output size can be specified as a single value (for both rows and cols, e.g. -n200) or comma-separated values (e.g. -n300,400).
    Use an Inf value to keep a dimension unchanged (e.g., -n50,inf).
    Use a NaN value to keep the aspect ratio unchanged (e.g., -n50, nan).

  15. The option -x<val> to set maximum output image size (bitmap formats only).
    The output size can be specified as a single value (for both rows and cols, e.g. -x200) or comma-separated values (e.g. -x300, 400).
    Use an Inf value to keep a dimension unchanged (e.g. -x50,inf).
    Use a NaN value to keep aspect ratio unchanged (e.g. -x50,nan).

  16. The option -s<val> to scale output image to a specific size (bitmap formats only).
    The fixed size can be specified as a single value (for rows=cols) or comma-separated values (e.g. -s300,400).
    Each value can be a scalar integer (signifying pixels) or percentage (e.g., -s125%).
    The scaling is done last, after any other cropping/rescaling due to other params.

  17. The option -append indicates that if the file already exists the figure is to be appended as a new page, instead of being overwritten (default).
    PDF, TIF & GIF output formats only (multi-image GIF = animated).

  18. The option -bookmark to indicate that a bookmark with the name of the figure is to be created in the output file (PDF format only).

  19. The option -clipboard to save the output as an image on the system clipboard.

  20. The option -clipboard<:format> - copies to clipboard in the specified format: image (default), bitmap, emf, or pdf.
    Note that Only -clipboard (or -clipboard:image, which is the same) applies savefig parameters such as cropping, padding, etc:

    1. The option -clipboard:image creates a bitmap image using savefig processing.
    2. The option -clipboard:bitmap creates a bitmap image as-is (no auto-cropping etc.).
    3. The option -clipboard:emf is vector format without auto-cropping; Windows-only.
    4. The option -clipboard:pdf is vector format without cropping; not universally supported.

  21. The option -d<gs_option> to indicate a ghostscript setting. For example, -dMaxBitmap=0 or -dNoOutputFonts (Ghostscript 9.15+).

  22. The option -depsc - option to use EPS level-3 rather than the default level-2 print device. This solves some bugs with MATLAB's default -depsc2 device such as discolored subplot lines on images (vector formats only).

  23. The option -metadata <metaDataInfo> - adds the specified meta-data information to the exported file (PDF format only). metaDataInfo must be either a struct or a cell array with pairs of values: {'fieldName', fieldValue, ...}.
    Common metadata fields: Title, Author, Creator, Producer, Subject, Keywords.

  24. The option -update - downloads and installs the latest version of savefig.

  25. The option -version returns the current savefig version without any figure export.

  26. The option -nofontswap - avoids font swapping. Font swapping is automatically done in vector formats (only): 11 standard MATLAB fonts are replaced by the original figure fonts. This option prevents this.

  27. The option -font_space <char> sets a spacer character for font-names that contain spaces used by EPS/PDF. Default: ''.

  28. The option -linecaps creates rounded line-caps (vector formats only).

  29. The option -noinvert avoids setting figure's InvertHardcopy property to 'off' during output (this solves some problems of empty outputs).

  30. The option -preserve_size preserves the figure's PaperSize property in output file (PDF/EPS formats only; default is to not preserve it).

  31. The option -options <optionsStruct> - format-specific parameters as defined in MATLAB documentation of the imwrite function, contained in a struct under the format name. For example, to specify the JPG Comment parameter, pass a struct such as this: options.JPG.Comment='abc'. Similarly, options.PNG.BitDepth=4. Only used by PNG, TIF, JPG, and GIF output formats.
    Options can also be specified as a cell array of name-value pairs, e.g., {'BitDepth', 4, 'Author', 'Yair'} - these options will be used by all supported output formats of the savefig command.

  32. The option -silent to avoid various warning and informational messages, such as version update checks, transparency or renderer issues, etc.

  33. The option -notify to notify the user when export is done, in both a console message and a popup dialog (allow opening the exported file/folder).

  34. The option -regexprep <old> <new> - replaces all occurrences of <old> (a regular expression string or array of strings; case-sensitive), with the corresponding <new> string(s), in EPS/PDF files (only). See regexp function doc.
    Warning: invalid replacement can make your EPS/PDF file unreadable!

  35. The option -toolbar - adds an interactive export button to the figure toolbar.

  36. The option -menubar - adds an interactive export menu to the figure menubar.

  37. The option -contextmenu - adds interactive export menu to figure context-menu (right-click)

  38. The option handle - handle of the figure, axes, or uipanels (can be an array of handles but all the objects must be in the same figure) to be exported.
    Default: gcf (handle of the current figure).

  39. The option figName - name (title) of the figure to export (e.g., 'Figure 1' or 'My fig').
    Overridden by handle (if specified); Default: current figure.
Returns
imageData : The output image cube of type uint8 of shape [M, N, C] containing the exported figure data.
alpha : The output image matrix of shape [M, N] of alpha-matte values in the range [0, 1] for the case of transparent background.

Example usage ⛓

pm.vis.figure.savefig(); % export the current figure with the default name.
pm.vis.figure.savefig("gridplot.pdf") % export figure to the specified PDF file.
pm.vis.figure.savefig("gridplot.png", "-m4 -transparent") % export a large png plot of magnitude 4 with transparency.
