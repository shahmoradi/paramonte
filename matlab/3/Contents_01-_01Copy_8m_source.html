3%> What is ParaMonte?
4%> ==================
5%>
6%> ParaMonte is a serial/parallel library of Monte Carlo (**MC**) optimization, sampling, and integration routines
7%> for exploring arbitrary-dimensional mathematical density functions, in particular, the posterior distributions of
8%> Bayesian models in data science, machine learning, and scientific inference, as well as Machine Leaning (**ML**)
9%> algorithms for scientific inference.
10%>
11%> ParaMonte has been designed with the goal of unifying the
12%>
13%> + **automation** (of MC and ML simulations),
14%> + **user-friendliness** (of the library),
15%> + **accessibility** (from multiple programming environments),
16%> + **high-performance** (at runtime), and
17%> + **scalability** (across many parallel processors).
18%>
19%> For more information on the installation, usage, and examples, visit:
20%>
21%> https://www.cdslab.org/paramonte
22%>
23%> To get started with the library, simply create an instance of the @paramonte
24%> class to access the routines and functionalities available in the library.
25%> Before doing so, ensure to add the library path to your MATLAB session,
26%> as done below enclosed between the two comment lines,
27%>
28%> pmlibRootDir = './'; % Change this to the directory containing the ParaMonte MATLAB package.
29%> addpath(pmlibRootDir);
30%> pm.version
31%> doc pm
32%>
33%> Modules
34%> -------
35%>
36%> The MATLAB equivalent of modules is called **package**.
37%> Similar to modules in other languages, MATLAB packages can be imported to the MATLAB environment
38%> or simply be used as namespace. For example,
39%>
40%> pm.lib.version()
41%>
42%> The ParaMonte MATLAB library currently contains the following modules:
43%>
44%> pm.array - module for array manipulation tasks.
45%> pm.introspection - module for MATLAB type introspection.
46%> pm.io - module for IO tasks.
47%> pm.lib - module for ParaMonte library information retrieval.
48%> pm.math - module for mathematical tasks.
49%> pm.matlab - module for MATLAB-specific tasks.
50%> pm.os - module for Operating System-specific tasks.
51%> pm.sampling - module for stochastic sampling of mathematical objective functions.
52%> pm.sort - module for sorting tasks.
53%> pm.str - module for string manipulation tasks.
54%> pm.sys - module for system-level operations.
55%> pm.timing - module for timing tasks.
56%> pm.vis - module for visualization tasks.
57%> pm.web - module for world-wide-web tasks.
58%>
59%> Additionally, the package contains the following folder that are meant to be used internally by the library.
60%>
61%> pm/auxil - Folder contains auxiliary information and functionalities required by the library.
62%> pm/lib - Folder contains the ParaMonte MATLAB lower-level shared libraries.
63%>
64%> Naming conventions
65%> ------------------
66%>
67%> The camelCase naming style is used throughout the entire ParaMonte MATLAB library, across
68%> all programming languages including C/C++/Fortran/Julia/MATLAB/Python/R.
69%>
70%> All simulation specifications start with a lowercase letter, including
71%> scalar/vector/matrix int, float, string, or Boolean variables.
72%>
73%> Occasionally, variable names representing a vector of values may be suffixed with "Vec".
74%>
75%> Occasionally, variable names representing a matrix of values may be suffixed with "Mat".
76%>
77%> Occasionally, variable names representing a list of values may be suffixed with "List".
78%>
79%> Frequently, functions or class methods begin with a lowercase verb.
80%>
81%> Significant attempt has been made to name all Boolean variables like a logical
82%> proposition that is evaluated to either true or false, set by the user.
