1%> \brief
2%> Return a copy of the input ``object``
3%> whose property (or field) values are overwritten
4%> with the corresponding (key, val) pairs in the
5%> input ``hashmap`` cell array.
6%>
7%> \note
8%> This functionality is useful for converting
9%> the variadic arguments of functions to the
10%> corresponding property values of objects
11%> with the functions.
12%>
13%> \param[in] hashmap : The input cell array of even number of elements
14%> containing the field names and values of the input ``object``
15%> as ``(key, val)`` pairs stored sequentially as the cell elements.
16%> All keys in the input ``hashmap`` must already exist as
17%> fields/properties of the input object.
18%>
19%> \param[in] object : The input scalar MATLAB struct
20%> or object whose property/field values
21%> are to be overwritten with the corresponding
22%> values from the input hashmap cell.
23%>
24%> \param[in] insensitive : The input scalar MATLAB logical.
25%> If ``false``, then keys within the input ``hashmap`` will be matched
26%> against the input ``object`` properties(or fields) case-sensitively.
27%> (**optional**, default = ``false``)
28%>
29%> \param[in] extensible : The input scalar MATLAB logical.
30%> If ``true``, then keys in the input ``hashmap`` that are missing in the
31%> input ``object`` properties(or fields) will be added to output ``objnew``.
32%> This functionality requires the input ``object`` to be either a MATLAB ``struct``
33%> or an object whose ultimate superclass is the MATLAB ``handle`` class,
34%> which allows adding new properties via ``addprop()`` method.
35%> (**optional**, default = ``false``)
36%>
37%> \param[in] recursive : The input scalar MATLAB logical.
38%> If ``true``, then key-val pairs in the input ``hashmap`` that match struct-cell
39%> or match object-cell pattern will be also converted recursively like the parent
40%> input ``object`` until all sub-components are obtained recursively.
41%> (**optional**, default = ``false``)
42%>
43%> \return
44%> objnew : The output copy of the input object whose
45%> property/field values are overwritten with the
46%> corresponding values from the input hashmap cell.
47%>

61%>
62%> \endcode
63%> \example{hash2comp}
