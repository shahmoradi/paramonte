1%> \brief
2%> Return a scalar MATLAB logical that is ``true`` if and
3%> only if the deletion of the input file fails,
4%> otherwise, return ``false``.
5%>
6%> \details
7%> This function is a simple wrapper around the MATLAB function ``delete()``
8%> It is primarily meant to bring consistency to messaging file deletions
9%> if they fail. Such consistency is particularly required on Windows
10%> systems as the OS tends to lock file ownership to one application
11%> causing deletions to fail frequently.
12%>
13%> \param[in] file : The input scalar MATLAB string,
14%> containing the file path to be deleted.
15%>
16%> \param[in] desc : The input scalar MATLAB string, containing a descriptive message
17%> to be printed on the MATLAB command line if the deletion task fails.
18%> The input ``desc``, if not empty, will be added to the following template
19%> before being displayed:<br>
20%> "Failed to delete <desc> from the local disk. File may be protected."<br>
21%> If the input ``desc`` is empty ``[]`` or empty string ``""``, a default
22%> value ``"the requested file"`` for ``desc`` will be added to the template.<br>
23%> (**optional**. If missing, no warning will be displayed upon failure.)
24%>
25%> \return
26%> `failed` : The output scalar MATLAB logical that is ``true`` if and
27%> only if the deletion of the input file fails,
28%> otherwise, return ``false``.
29%>
30%> \interface{rmfile}
31%> \code{.m}
32%>
33%> failed = pm.sys.path.rmfile(file)
34%> failed = pm.sys.path.rmfile(file, desc)
35%>
36%> \endcode
37%> \final{rmfile}
