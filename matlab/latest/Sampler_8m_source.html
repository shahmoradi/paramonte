33 %> \param checked : Optional scalar MATLAB logical.
34 %> If set to ``true``, then the checked (verified) versions of the
35 %> ParaMonte MATLAB shared libraries will be used for simulations.
36 %> This means that most function calls and computations will be
37 %> checked at runtime to ensure correctness, at the cost of
38 %> noticeably slower runtime simulation speeds.<br>
39 %> Set this option to ``true`` only for diagnostics, testing, and development.
40 %> If set to empty, the sampler will first search for the non-checked version
41 %> and if none found, will search for a checked version of the library.
42 %>
43 checked = [];
44 %>
45 %> \param mpiname : Optional scalar MATLAB string representing the (vendor) name of the
46 %> MPI runtime library. The following three library names are supported:<br>
47 %> - "impi" : The Intel MPI runtime library (Windows/Linux).
48 %> - "mpich" : The MPICH MPI runtime library (Linux/macOS).
49 %> - "openmpi" : The OpenMPI runtime library (Linux/macOS).<br>
50 %> By default, you should set this argument to ``pm.lib.mpi.choice()``
51 %> to enable the right preferred choice of MPI-enabled ParaMonte libraries.
52 %> If for any reason, you want to use a non-preferred non-default MPI library,
53 %> you can set the MPI library name explicitly as in the above list.<br>
54 %> Beware that you must launch MATLAB with an ``mpiexec`` binary that
55 %> comes from the same MPI library vendor specified for ``mpiname``.
56 %> Note that the MPI-parallel ParaMonte shared libraries
57 %> corresponding to the specified value for ``mpiname``
58 %> may not exist in the ParaMonte package.
59 %> For example,<br>
60 %> - On Windows, only ParaMonte library
61 %> builds with Intel MPI are supported.<br>
62 %> - On Linux, the ParaMonte library builds with
63 %> all MPI libraries listed above are supported,
64 %> but there is no guarantee of their availability in
65 %> the package. Only the ParaMonte library builds with
66 %> Intel MPI (``"impi"``) are guaranteed to exist,
67 %> unless user builds the ParaMonte MATLAB package
68 %> from GitHub source for the desired MPI library.<br>
69 %> - On Darwin (macOS), only the ParaMonte shared library
70 %> builds with MPICH and OpenMPI libraries are supported,
71 %> but there is no guarantee of their availability in the
72 %> package. Only the ParaMonte library builds with OpenMPI
73 %> (``"openmpi"``) are guaranteed to exist, unless user
74 %> builds the ParaMonte MATLAB package from GitHub
75 %> source for the desired specified MPI library.<br>
76 %> \warning
77 %> USE THIS OPTIONAL ARGUMENT ONLY IF YOU KNOW ITS RAMIFICATIONS.
78 %> This option should rarely be needed in any normal simulation..<br>
79 %> The default value for ``mpiname`` is an empty string,
80 %> implying no use of MPI-enabled ParaMonte library routines.
81 %>
82 mpiname = "";
83 %>
84 %> \param silent : Optional logical (Boolean) indicator which is ``false`` by default.
85 %> If it is set to ``true``, it will silence all output postprocessing messages.
86 %>
87 %> \note
88 %> Setting ``mpiname`` to a non-empty string
89 %> will automatically set ``silent`` to ``true``.
