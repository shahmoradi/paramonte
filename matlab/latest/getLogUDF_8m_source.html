1%> \brief
2%> Return the natural logarithm of the Unnormalized Density Function (UDF)
3%> of the inverse of the 2-dimensional modified Himmelblau function.
4%>
5%> \details
6%> Himmelblau's function is a multi-modal function, used to test
7%> the performance of optimization algorithms. The function is defined by:
8%> \f{equation}
9%> H(x, y) = (x^{2} + y - 11)^{2} + (x + y^{2} - 7)^{2} ~.
10%> \f}
11%>
12%> It has one local maximum at \f$x = -0.270845\f$ and \f$y = -0.923039\f$ where \f$H(x, y) = 181.617\f$,
13%> and four identical local minima:
14%> \f{equation}
15%> H(3.0,2.0) = 0.0 ~,
16%> H(-2.805118, 3.131312)=0.0 ~,
17%> H(-3.779310, -3.283186)=0.0 ~,
18%> H(3.584428, -1.848126)=0.0 ~.
19%> \f}
20%>
21%> The function is named after David Mautner Himmelblau (1924â€“2011), who introduced it.<br>
22%> The locations of all the minima can be found analytically.<br>
23%>
24%> This MATLAB function returns a modification of the Himmelblau function as a density
25%> function suitable for testing sampling algorithms (or stochastic maximizers):
26%> \f{equation}
27%> f(x, y, \epsilon) = \frac{1}{H(x, y) + \epsilon} ~.
28%> \f}
29%>
30%> \param[in] x : The input scalar or array of the same rank and shape as
31%> other input array-like arguments of type MATLAB ``double``,
32%> representing the x-component of the state within the domain
33%> of Himmelblau density at which the density value must be computed.<br>
34%> \param[in] y : The input scalar or array of the same rank and shape as
35%> other input array-like arguments of type MATLAB ``double``,
36%> representing the y-component of the state within the domain
37%> of Himmelblau density at which the density value must be computed.<br>
38%>
39%> \param[in] epsilon : The input positive scalar or array of the same rank and shape as
40%> other array-like input arguments of type MATLAB ``double``,
41%> representing the value to be added to the inverse
42%> of the Himmelblau function.<br>
43%> Increasingly smaller values of ``epsilon`` will yield pointier densities.<br>
44%> Increasingly larger values of ``epsilon`` will yield flatter densities.<br>
45%> (**optional**, default = ``1``)
46%>
47%> \return
48%> `logUDF` : The output Unnormalized Density Function (UDF) of the
49%> Himmelblau density at the specified input ``state``.
50%>
51%> \interface{getLogUDF}
52%> \code{.m}
53%>
54%> logUDF = pm.stats.himmelblau.getLogUDF(state);
55%> logUDF = pm.stats.himmelblau.getLogUDF(state, epsilon);
56%>
57%> \endcode
