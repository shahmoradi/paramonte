310 %> \brief
311 %> Export the current figure to the specified external file.
312 %>
313 %> \details
314 %> This method internally uses the venerable ``export_fig`` MATLAB package.<br>
315 %> As such, it accepts all arguments that the ``export_fig()`` function accepts.<br>
316 %> If no optional argument is present, then a default set of options determined
317 %> by the ``export_fig`` library will be used.<br>
318 %>
319 %> \param[in] file : The input scalar MATLAB string or char vector containing
320 %> the path to the external file that will contain the visualization.
321 %> The specified file extension determines its type (e.g., ``.pdf``, ``.png``).<br>
322 %> If no file extension is specified, then the default ``.png`` file extension is used.<br>
323 %> (**optional**. If ``file`` is missing or empty, first, the ``fileName`` component of
324 %> the ``figure`` property of the parent object will be used as the filename if not empty.<br>
325 %> Otherwise, the figure will be exported to a file in the current working directory of
326 %> MATLAB with name ``figure`` suffixed by a unique number and ``.png`` extension.)
327 %>
328 %> \param[in] varargin : The following optional flags are also acceptable as input string arguments.
329 %> This method internally uses the venerable ``export_fig`` MATLAB package.
330 %> As such, it accepts all arguments that the ``export_fig()`` function accepts.
331 %> If no optional argument is present, then a default set of options determined
332 %> by the ``export_fig`` library will be used.<br>
333 %>
334 %> The recommended set of optional flags for PNG file formats is ``"-m4 -transparent"``.<br>
335 %> <ol>
336 %> <li> ``"-<format>"``
337 %>
338 %> The input scalar (or variadic array of) MATLAB string(s)
339 %> containing the output file extension(s). Options include:
340 %>
341 %> <ol>
342 %> <li> ``'-pdf'``
343 %> <li> ``'-eps'``
344 %> <li> ``'-emf'``
345 %> <li> ``'-svg'``
346 %> <li> ``'-png'``
347 %> <li> ``'-tif'``
348 %> <li> ``'-jpg'``
349 %> <li> ``'-gif'``
350 %> <li> ``'-bmp'``
351 %> </ol>
352 %>
353 %> Multiple formats can be specified, without restriction.<br>
354 %> For example:
355 %>
356 %> \code{.m}
357 %>
358 %> savefig('-jpg', '-pdf', '-png')
359 %>
360 %> \endcode
361 %>
362 %> Note that ``'-tif'`` and ``'-tiff'`` are equivalent.<br>
363 %>
364 %> <li> ``"-transparent"``
365 %>
366 %> The input option indicating that the figure background is to
367 %> be made transparent (PNG, PDF, TIF, EPS, EMF formats only).<br>
368 %> Specifying this option also activates ``"-noinvert"``.<br>
369 %>
370 %> <li> ``"-nocrop"``
371 %>
372 %> The input option indicating that empty margins should not be cropped.<br>
373 %>
374 %> <li> ``"-c[<val>,<val>,<val>,<val>]"``
375 %>
376 %> The input option indicating the crop amounts. It must be
377 %> a 4-element vector of numeric values: ``[top, right, bottom, left]``
378 %> where NaN/Inf indicates auto-cropping, 0 means no cropping, any
379 %> other value means cropping in pixel amounts. e.g. ``'-c7,15,0,NaN'``
380 %> Note that this option is not supported by SVG and EMF formats.<br>
381 %>
382 %> <li> ``"-p<val>"``
383 %>
384 %> The input option to pad a border of width ``val`` to exported files,
385 %> where ``val`` is either a relative size with respect to cropped image
386 %> size (i.e. ``p = 0.01`` adds a ``1%`` border). For EPS & PDF formats,
387 %> ``val`` can also be integer in units of ``1/72`` inch points (``abs(val) > 1``).<br>
388 %> ``val`` can be positive (padding) or negative (extra cropping).<br>
389 %> If used, the ``-nocrop`` flag will be ignored, i.e. the image will
390 %> always be cropped and then padded. Default: ``0`` (i.e. no padding).<br>
391 %> Note: this option is not supported by SVG and EMF formats.<br>
392 %>
393 %> <li> ``"-m<val>"``
394 %>
395 %> The input option ``val`` indicates the factor to magnify the figure
396 %> dimensions when generating bitmap outputs (does not affect vector formats).<br>
397 %> Default: '-m1' (i.e. val=1). Note: val~=1 slows down savefig.<br>
398 %>
399 %> <li> ``"-r<val>"``
400 %>
401 %> The input option ``val`` indicates the resolution (in pixels per inch)
402 %> to export bitmap and vector outputs, without changing dimensions of
403 %> the on-screen figure. Default: '-r864' (for vector output only).<br>
404 %> Note: -m option overides -r option for bitmap exports only.<br>
405 %>
406 %> <li> ``"-native"``
407 %>
408 %> The input option indicating that the output resolution (when
409 %> outputting a bitmap format) should be such that the vertical
410 %> resolution of the first suitable image found in the figure is
411 %> at the native resolution of that image. To specify a particular
412 %> image to use, give it the tag 'export_fig_native'.<br>
413 %> Notes: This overrides any value set with the -m and -r options.<br>
414 %> It also assumes that the image is displayed front-to-parallel
415 %> with the screen. The output resolution is approximate and
416 %> should not be relied upon. Anti-aliasing can have adverse
417 %> effects on image quality (disable with the -a1 option).<br>
418 %>
419 %> <li> ``"-a1, -a2, -a3, -a4"``
420 %>
421 %> The input option indicating the amount of anti-aliasing (AA) to
422 %> use for bitmap outputs, when GraphicsSmoothing is not available.<br>
423 %> '-a1'=no AA; '-a4'=max. Default: 3 for HG1, 1 for HG2.<br>
424 %>
425 %> <li> ``"-<renderer>"``
426 %>
427 %> The input option to force a particular renderer (painters, opengl or
428 %> [in R2014a or older] zbuffer). Default value: opengl for bitmap
429 %> formats or figures with patches and/or transparent annotations;
430 %> painters for vector formats without patches/transparencies.<br>
431 %>
432 %> <li> ``"-<colorspace>"``
433 %>
434 %> The input option indicating which colorspace color figures should
435 %> be saved in: RGB (default), CMYK or gray. Usage example: '-gray'.<br>
436 %> Note: CMYK is only supported in PDF, EPS and TIF formats.<br>
437 %>
438 %> <li> ``"-q<val>"``
439 %>
440 %> The input option to vary bitmap image quality (PDF, EPS, JPG formats only).<br>
441 %> A larger val, in the range 0-100, produces higher quality and
442 %> lower compression. val > 100 results in lossless compression.<br>
443 %> Default: '-q95' for JPG, ghostscript prepress default for PDF,EPS.<br>
444 %> Note: lossless compression can sometimes give a smaller file size
445 %> than the default lossy compression, depending on the image type.<br>
446 %>
447 %> <li> ``"-n<val>"``
448 %>
449 %> The input option to set minimum output image size (bitmap formats only).<br>
450 %> The output size can be specified as a single value (for both rows
451 %> & cols, e.g. -n200) or comma-separated values (e.g. -n300,400).<br>
452 %> Use an Inf value to keep a dimension unchanged (e.g. -n50,inf).<br>
453 %> Use a NaN value to keep aspect ratio unchanged (e.g. -n50,nan).<br>
454 %>
455 %> <li> ``"-x<val>"``
456 %>
457 %> The input option to set maximum output image size (bitmap formats only).<br>
458 %> The output size can be specified as a single value (for both rows
459 %> & cols, e.g. -x200) or comma-separated values (e.g. -x300,400).<br>
460 %> Use an Inf value to keep a dimension unchanged (e.g. -x50,inf).<br>
461 %> Use a NaN value to keep aspect ratio unchanged (e.g. -x50,nan).<br>
462 %>
463 %> <li> ``"-s<val>"``
464 %>
465 %> The input option to scale output image to specific size (bitmap formats only).
466 %> The fixed size can be specified as a single value (for rows=cols) or
467 %> comma-separated values (e.g. -s300,400). Each value can be a scalar
468 %> integer (signifying pixels) or percentage (e.g. -s125%). The scaling
469 %> is done last, after any other cropping/rescaling due to other params.<br>
470 %>
471 %> <li> ``"-append"``
472 %>
473 %> The input option indicating that if the file already exists the figure is to
474 %> be appended as a new page, instead of being overwritten (default).<br>
475 %> PDF, TIF & GIF output formats only (multi-image GIF = animated).<br>
476 %>
477 %> <li> ``"-bookmark"``
478 %>
479 %> The input option to indicate that a bookmark with the name of the
480 %> figure is to be created in the output file (PDF format only).<br>
481 %>
482 %> <li> ``"-clipboard"``
483 %>
484 %> The input option to save output as an image on the system clipboard.<br>
485 %>
486 %> <li> ``"-clipboard<:format>"``
487 %>
488 %> The input copies to clipboard in the specified format:
489 %> image (default), bitmap, emf, or pdf.<br>
490 %> Notes that only ``-clipboard`` (or ``-clipboard:image``, which is the same)
491 %> applies export_fig parameters such as cropping, padding etc.
492 %> <ol>
493 %> <li> ``-clipboard:image`` create a bitmap image using export_fig processing
494 %> <li> ``-clipboard:bitmap`` create a bitmap image as-is (no auto-cropping etc.)
495 %> <li> ``-clipboard:emf`` is vector format without auto-cropping; Windows-only.
496 %> <li> ``-clipboard:pdf`` is vector format without cropping; not universally supported
497 %> </ol>
498 %>
499 %> <li> ``"-d<gs_option>"``
500 %>
501 %> The input option to indicate a ghostscript setting. For example,
502 %> ``-dMaxBitmap=0`` or ``-dNoOutputFonts`` (Ghostscript 9.15+).<br>
503 %>
504 %> <li> ``"-depsc"``
505 %>
506 %> The input option to use EPS level-3 rather than the default level-2 print
507 %> device. This solves some bugs with MATLAB's default -depsc2 device
508 %> such as discolored subplot lines on images (vector formats only).<br>
509 %>
510 %> <li> ``"-metadata <metaDataInfo>"``
511 %>
512 %> The input adds the specified meta-data information to the
513 %> exported file (PDF format only). metaDataInfo must be either a struct
514 %> or a cell array with pairs of values: ``{'fieldName', fieldValue, ...}``.<br>
515 %> Common metadata fields: Title,Author,Creator,Producer,Subject,Keywords<br>
516 %>
517 %> <li> ``"-nofontswap"``
518 %>
519 %> The input option to avoid font swapping. Font swapping is automatically
520 %> done in vector formats (only): 11 standard MATLAB fonts are
521 %> replaced by the original figure fonts. This option prevents this.<br>
522 %>
523 %> <li> ``"-font_space <char>"``
524 %>
525 %> The input option to set a spacer character for font-names that
526 %> contain spaces, used by EPS/PDF. Default: ''<br>
527 %>
528 %> <li> ``"-linecaps"``
529 %>
530 %> The input option to create rounded line-caps (vector formats only).<br>
531 %>
532 %> <li> ``"-noinvert"``
533 %>
534 %> The input option to avoid setting figure's InvertHardcopy property to
535 %> 'off' during output (this solves some problems of empty outputs).<br>
536 %>
537 %> <li> ``"-preserve_size"``
538 %>
539 %> The input option to preserve the figure's PaperSize property in output
540 %> file (PDF/EPS formats only; default is to not preserve it).<br>
541 %>
542 %> <li> ``"-options <optionsStruct>"``
543 %>
544 %> The input format-specific parameters as defined in MATLAB's
545 %> documentation of the ``imwrite`` function, contained in a struct under
546 %> the format name. For example to specify the JPG Comment parameter,
547 %> pass a struct such as this: ``options.JPG.Comment='abc'``.<br>
548 %> Similarly, ``options.PNG.BitDepth = 4``.<br>
549 %> Only used by PNG, TIF, JPG, GIF output formats.<br>
550 %> Options can also be specified as a cell array of name-value pairs,
551 %> e.g. ``{'BitDepth', 4, 'Author', 'Yair'}``. These options will be used
552 %> by all supported output formats of the ``pm.vis.figure.savefig()`` function.<br>
553 %>
554 %> <li> ``"-silent"``
555 %>
556 %> The input option to avoid various warning and informational messages,
557 %> such as version update checks, transparency or renderer issues, etc.<br>
558 %>
559 %> <li> ``"-notify"``
560 %>
561 %> The input option to notify the user when export is done, in both a console
562 %> message and a popup dialog (allow opening the exported file/folder).<br>
563 %>
564 %> <li> ``"-regexprep <old> <new>"``
565 %>
566 %> The input replaces all occurrences of <old> (a regular expression
567 %> string or array of strings; case-sensitive), with the corresponding
568 %> new string(s), in EPS/PDF files (only). See regexp function's doc.<br>
569 %> Warning: invalid replacement can make your EPS/PDF file unreadable!<br>
570 %>
571 %> <li> ``"-toolbar"``
572 %>
573 %> The input adds an interactive export button to the figure's toolbar<br>
574 %>
575 %> <li> ``"-menubar"``
576 %>
577 %> The input adds an interactive export menu to the figure's menubar<br>
578 %>
579 %> <li> ``"-contextmenu"``
580 %>
581 %> The input adds interactive export menu to figure context-menu (right-click)<br>
582 %>
583 %> <li> ``"handle"``<br>
584 %>
585 %> The input handle of the figure, axes or uipanels (can be an array of handles
586 %> but all the objects must be in the same figure) to be exported.<br>
587 %> The default is ``gcf`` (handle of current figure).<br>
588 %>
589 %> <li> ``"figName"``
590 %>
591 %> The input name (title) of the figure to export (e.g. ``'Figure 1'`` or ``'My fig'``).<br>
592 %> Overridden by handle (if specified)<br>
593 %> The default is the current figure.<br>
594 %>
595 %> </ol>
