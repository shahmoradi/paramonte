3 @brief
4 Return the MPI image count and the current image ID (e.g., MPI rank + 1)
5 if the application has been launched via the Intel MPI ``mpiexec`` launcher.<br>
6
7 @details
8 This function attempts to detect the invocation of the current MATLAB
9 session via an Intel MPI ``mpiexec`` binary MPI launcher instance.<br>
10 The runtime detection is achieved by checking for runtime environment
11 variables that the Intel MPI ``mpiexec`` binary executable defines upon launch.<br>
12
13 Specifically, Intel MPI provides a long informal list of
14 environment variables that are defined on every MPI process.<br>
15 These variables may not be defined on all platforms.<br>
16 Some are only defined at runtime, while others are defined at compile-time.<br>
17
18 <ol>
19 <li> On **Unix systems**, the best approach to investigate the Intel-specific runtime variables
20 is to launch the Intel ``mpiexec`` binary executable with the ``printenv`` Unix command.<br>
21 @code{.sh}
22 mpiexec -np 1 printenv
23 @endcode
24 To see the compile-time variables and compare them with the runtime list from the above, try:<br>
25 @code{.sh}
26 printenv
27 @endcode
28 <li> On **Windows systems**, the best approach to investigate the Intel-specific runtime variables
29 is to launch the Intel ``mpiexec`` binary executable with the ``set`` Windows batch command.<br>
30 @code{.bat}
31 mpiexec -localonly -n 1 cmd /k set
32 @endcode
33 To see the compile-time variables and compare them with the runtime list from the above, try:<br>
34 @code{.bat}
35 set
36 @endcode
37 </ol>
38
39 Note that all Intel-MPI-specific environment variables are prefixed with ``I_MPI_``.<br>
40 The most persistent environment variables for Intel ``mpiexec`` binary appear to be the following:<br>
41
42 <ol>
43 <li> ``I_MPI_INFO_STATE`` : The status code for the Intel MPI runtime (e.g., ``0``).<br>
44 <li> ``I_MPI_INFO_BRAND`` : The processor brand (e.g., ``12th Gen Intel(R) Core(TM) i9-12950HX``).<br>
45 <li> ``PMI_RANK`` : The rank of the current process in the current MPI communication world.<br>
46 This environment variable is common to all Hydra-based MPI libraries, such as MPICH and Intel MPI libraries.<br>
47 <li> ``PMI_SIZE`` : The size (i.e., number of images/processes) of the current MPI communication world.<br>
48 This environment variable is common to all Hydra-based MPI libraries, such as MPICH and Intel MPI libraries.<br>
49 <li> ``MPI_LOCALRANKID`` : The **local** rank of the current process relative to the local host.<br>
50 This environment variable is common to all Hydra-based MPI libraries, such as MPICH and Intel MPI libraries.<br>
51 <li> ``MPI_LOCALNRANKS`` : The size (i.e., number of images/processes) of the current MPI communication on the local host.<br>
52 This environment variable is common to all Hydra-based MPI libraries, such as MPICH and Intel MPI libraries.<br>
53 </ol>
54
55 Therefore, this function first attempts to detect the Hydra process manager.<br>
56 If identified, the function will then attempt to identify the Intel ``mpiexec`` runtime variables.<br>
57
58 @return
59 ``nproc`` : The output scalar MATLAB non-negative whole number containing
60 the number of MPI images in the current invocation of the
61 Intel MPI ``mpiexec`` binary MPI launcher.<br>
62 An output value of ``0`` implies the algorithm failed to detect
63 a ParaMonte-compatible Intel MPI library or failed to infer the image count.<br>
64 ``rankp1`` : The output scalar MATLAB non-negative whole number containing
65 the image ID (e.g., MPI rank + 1) of the current MPI image in
66 the current invocation of the Intel MPI ``mpiexec`` binary MPI launcher.<br>
67 An output value of ``0`` implies the algorithm failed to detect
68 a ParaMonte-compatible MPI library or failed to infer the image count.<br>
69 Note that the image ID always starts at ``1``, unlike the MPI rank.<br>
70 The argument ``rankp1`` stands for ``rank + 1``.<br>
71 An output value of ``0`` implies the algorithm failed to detect
72 a ParaMonte-compatible Intel MPI library or failed to infer the image count.<br>
73
