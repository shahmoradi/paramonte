Return a scalar Python logical that is True if and only if the deletion of files or directories matching the input pattern fails, otherwise, return False.


This function is a simple wrapper around the Python intrinsic functions for deletion.
It is primarily meant to bring consistency to messaging file/folder deletions if they fail.
Such consistency is particularly required on Windows systems as the OS tends to lock file ownership to one application causing deletions to fail frequently.
This function also adds the capability to delete file or folder patterns through glob intrinsic module.

Parameters
[in]pattern: The input scalar Python string, containing the path pattern of the files or folders to be deleted.
[in]desc: The input scalar Python string, containing a descriptive message to be printed on the Python command line if the deletion task fails.
The input desc, if not empty, will be added to the following template before being displayed:
"Failed to delete <desc> from the local disk. Path may be protected."
If the input desc is an empty string "", a default value "an instance of the requested path pattern" for desc will be added to the template.
If the input argument is missing or set to None, no warning message will be displayed.
(optional. default = None)
Returns
failed : The output scalar Python logical that is True if and only if the deletion of the input pattern fails, otherwise, it is False.
